أنواع البيانات في Python

كيف تتعامل لغة بايثون مع الكائنات في الذاكرة؟

🔒 الكائنات غير المتغيرة (Immutable)

في بايثون، بمجرد إنشاء هذه الكائنات، لا يمكن تغيير حالتها أو محتواها. أي محاولة تعديل تؤدي لإنشاء كائن جديد بمعرف (ID) جديد.

Numbers int, float, complex
Strings str
Tuples tuple
# اختبار الهوية في بايثون
x = 5
print(id(x)) # مثلاً: 14070
x = x + 1
print(id(x)) # سيتغير الـ ID حتماً!

📝 الكائنات المتغيرة (Mutable)

تسمح بايثون بتغيير محتوى هذه الكائنات "في مكانها" (In-place). هذا يعني أن الـ ID الخاص بالكائن يبقى ثابتاً حتى بعد التعديل.

Lists list
Dictionaries dict
Sets set
# تعديل القائمة في مكانها
my_list = [1, 2]
print(id(my_list))
my_list.append(3)
print(id(my_list)) # نفس الـ ID تماماً

ملاحظات هامة للمبرمجين (Python Tips):

  • تحذير القيم الافتراضية: لا تستخدم أبداً كائنات متغيرة (مثل القوائم) كقيم افتراضية في تعريف الدوال؛ لأنها تشارك نفس الكائن عبر جميع الاستدعاءات.
  • دالة Copy: عند التعامل مع القوائم المتداخلة، استخدم copy.deepcopy() لتجنب تعديل البيانات الأصلية بالخطأ.
  • مفاتيح القواميس: بايثون تشترط أن يكون مفتاح القاموس Immutable دائماً.
